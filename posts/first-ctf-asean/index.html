Challenges for BlockChain technology (1/5) (153 solves)

Description

A healthcare institution implemented a blockchain-based system for storing patient medical records to ensure integrity of medical data. They developed the smart contract below to store and manage patient information on the Ethereum blockchain.

pragma solidity ^0.8.0;

contract MedicalRecords {
	struct MedicalData { 
		string patientName; 
		uint256 patientAge; 
		string patientCondition; 
		string treatment; 
	}

	mapping(uint256 => MedicalData) public records;
	event RecordAdded(uint256 patientId, string patientName);

	function addRecord(uint256 patientId, string memory patientName, uint256 patientAge, string memory patientCondition, string memory treatment) public {
		records[patientId] = MedicalData(patientName, patientAge, patientCondition, treatment);
		emit RecordAdded(patientId, patientName);
	}

	function getRecord(uint256 patientId) public view returns (string memory, uint256, string memory, string memory) { 
		MedicalData memory record = records[patientId]; 
		return (record.patientName, record.patientAge, record.patientCondition, record.treatment); 
	} 
}

Is there any structure violating privacy in this smart contract? If so, provide the name of the structure.

All the Way BACnet - 1/4 (105 solves)

Attachments
Attach: bacnet.pcap
Description

Summary

BACnet is a data communications protocol for building automation and control networks. BACnet is an extremely common protocol seen in many different types of industries and these challenges will serve as an introduction to the BACnet protocol. All three of these challenges will utilize the attached bacnet.pcap network capture.

The BACnet protocol defines a number of services that are used to communicate between building devices. Two of the most common services are who-Is and i-Am requests which are used for device and object discovery within a BACnet network. A who-Is message is a broadcast message sent over a BACnet network and an i-Am message is the response to the broadcast, indicating to the sender its IP Address, BACnet ObjectIdentifier, and Vendor ID.

Challenge

What is the BACnet ObjectIdentifier’s Instance Number for the device with the Vendor ID of Cornell University.

Flag Format

BACnet ObjectIdentifier’s Instance Number. Example: 123

All the Way BACnet - 2/4 (48 solves)

Attachments
Attach: bacnet_2.pcap
Description

Summary

The two most common BACnet services are read-property and write-property. These services are used to read/write from/to BACnet devices. These services indicate what data they are wanting to interact with by specifying a PropertyIdentifier. There are many different PropertyIdentifiers specified within the BACnet protocol such as object-name, present-value, units, out-of-service, and event-state.

Challenge

Flag Format

BACnet Object Name. Example: Sensor_123456

Yes We CANbus - 1/3 (125 solves)

Attachments
Attach: canbus.pcap
Description

Summary

The Controller Area Network (CANbus) is a message/frame based standard designed to allow microcontrollers and devices to communicate with each other without a host computer or dedicated wiring. CANbus is most commonly used in Electronic Control Unit (ECU) communication. Unlike TCP/IP networks, there are no IP addresses, instead they send a CAN ID as part of each frame indicating the purpose/target of the data. Every device on a CANbus network receives every frame of data and then must decide if the data is relevant for them based on the CAN ID.

Challenge

Most of the CAN IDs in this network are very talkative, however, one CAN ID is used only once. What is this CAN ID?

Flag Format

CAN ID. Example: 0x1ab

Yes We CANbus - 2/3 (20 solves)

Description

Summary

CANbus frames have a maximum size of 8 bytes. Therefore, there is not a lot of room for encryption, authentication, or reliability checks. In the CANbus packet capture from the previous challenge, CAN IDs 0x130, 0x2ce, and 0x4d8 are all used to send commands and values to various components. However, there is a checksum calculated to ensure the data received is the same as the data sent.

Challenge

What would the checksum be if you wanted to send a packet with the following data? “50 73 79 64 75 63 6b”

Flag Format

Checksum byte. Example: ff
